What are we talking about?

  • Transitions: Simple, implicit animations between two values.
  • Transforms: Modifying elements along the X, Y and/or Z axis.

Why use CSS instead of JavaScript?

How browsers work*

*Simplified. A fuller explanation.

Why use CSS?

  • JavaScript engine isn't running calculations
  • JavaScript engine doesn't have to communicate with the layout engine
  • Can be hardware accelerated if the browser and hardware support it

Transition Syntax

selector{
    transition-property: width, left;
    transition-duration: 2s, 1s;
    transition-delay: 0, 500ms;
    transition-timing-function: ease, linear;
}

Shorthand Transition Syntax

selector{
    transition: width 2s 0 ease, left 1s 500ms linear;
}

transition-property

Which property or properties to animate.

  • transition-property: left;
  • transition-property: font-size, color;
  • transition-property: all;
  • transition-property: none;

transition-duration

  • transition-duration: 500ms
  • transition-duration: 2s, 3s
  • transition-duration: 0s

Time units are defined in the CSS Values and Units Module Level 3 spec.

transition-delay

Delays the start of the transition.

  • transition-delay: 0s
  • transition-delay: 500ms
  • transition-delay: 2s, 1s
  • transition-delay: -2s

Which properties can you transition?

Properties with interpolate-able values

Rough guide: if it can accept numeric values, it's probably transitionable!

  • lengths
  • percentages
  • colors

See the specification for an exact list.

Transition Events

Only one so far: transitionend

el.addEventListener('transitionend', func, false);

Is the event prefixed too?

Yes (except in Firefox).

  • Firefox: transitionend
  • Webkit: webkitTransitionEnd
  • Opera: oTransitionEnd
  • Microsoft: MSTransitionEnd

How Do I Determine Which Event To Listen For?

Use Modernizr

var endNames = {
    'WebkitTransition' : 'webkitTransitionEnd',
    'MozTransition'    : 'transitionend',
    'OTransition'      : 'oTransitionEnd',
    'msTransition'     : 'MSTransitionEnd'
}
transEnd = endNames[ Modernizr.prefixed('transition') ];

Combining transitions & transforms

Compound transforms

Transforms can be combined. Applied in the order listed.

p{
     transition: transform 500ms linear;
     transform: skew(40deg) rotate(-45deg) scale(.3);
}

But the cascade still applies

p.transformed{
    transform: skew(40deg) rotate(-45deg) scale(.3);     transform: skew(-50deg);
}

Apply a new transform with all functions

p.transformed{
     transform: skew(-50deg) rotate(-45deg) scale(.3);
}

What about the prefixes?

What about the prefixes?

Yes, you need to include the prefix when transitioning prefixed properties.

-webkit-transition: -webkit-transform 500ms linear;
   -moz-transition:    -moz-transform 500ms linear;
    -ms-transition:     -ms-transform 500ms linear;
     -o-transition:      -o-transform 500ms linear;
